I fuzzy set vengono utilizzati per definire operazioni sul linguaggio naturale.
Prima di definire un fuzzy set e le sue operazioni è pertanto necessario definire il contesto in cui li adoperiamo.
un fuzzy set viene utilizzato per modellare una variabile linguistica(attributo in un sistema fuzzy) che può essere ad esempio: quanto sono grandi gli appartamenti in centro a Milano. La variabile linguistica viene modellata tramite dei valori linguistici che sono soggettivi e possono essere ad esempio: piccolo, medio, grande.
Ad ognugno di questi valori linguistici verrà associata una membership function che permetterà di passare dal dominio linguistico a quello dei numeri, infatti un'appartamento potrà appartenere alle varie categorie piccolo,medio e grande in maniera diversa con valori reali tra 0 e 1. I fuzzy set infatti, utilizzano una logica a valori reali [0,1] e sono pertanto una generalizzazione della classica logica booleana.
Data quindi l'introduzione al contesto, un fuzzy set è un mapping dai valori del dominio del discreto X (o insieme universo) a valori [0,1] tramite una membership function µ.
Una membership function µ è una funzione che associa ad ogni elemento x appartenente a X un valore [0,1] che indicherà il valore di appartenenza dell'elemento x al fuzzy set.
La membership function deve rispettare alcune caratteristiche fondamentali:
1. deve avere valori di appartenenza simili per elementi simili;
2. deve avere come massimo valore di appartenenza 1 e come minimo valore di appartenenza 0. (0 indica la non appartenenza all'insieme).
Se utilizzassimo valori crisp come nella logica booleana avremmo solo gli elementi 0 e 1, che indicherebbero rispettivamente la totale non appartenenza all'insieme e la totale appartenenza all'insieme.
La funzione di membership può assumere significati diversi in base a contesti diversi:
nel contesto dei data analysis e classification la membership function assume valore di somiglianza, in questo caso i valori della membership function indicano quanto i due elementi sono simili tra loro, questo concetto è importantissimo ad esempio per il clustering.
Nel contesto di decision making la membership function assume significato di preferenza, ovvero, quanto preferiamo una soluzione rispetto ad un'altra.
Nel contesto del approximate reasoning, invece, la membership function assume il significato di possibilità, ovvero, µ(u) indica qual è la possibilità che l'elemento X abbia valore u.
In generale però la membership function indica un concetto di incertezza e non di probabilità.
Infatti i valori reali [0,1] indicano il grado di appartenenza all'insieme fuzzy e non la probabilità che essi appartengano o meno all'insieme.
I fuzzy set possono essere rappresentati tramite 2 principali rappresentazioni che sono in modo verticale oppure orizzontale.
Nel metodo verticale ad ogni elemento del fuzzy set viene dato un valore in base alla membership function, nel metodo orizzontale invece, venogno considerato solo gli elementi del discorso X che hanno una membership function maggiore di una certa soglia alfa. Questa rappresentazione viene anche chiamata alfa-cuts e viene utilizzata per la rappresentazione dei fuzzy set all'interno dei calcolatori, infatti, essi vengono rappresentati tramite catene di liste lineari in cui ogni coppia di elementi delimita gli estremi di un alpha-cut.
Altre definizioni utili per descrivere i fuzzy set sono:
1. Support: è un insieme booleano rappresentato da elementi del dominio del discroso che hanno un grado di membership non nullo
2. Core: è un insieme booleano rappresentato da elementi del dominio del discorso che hanno un grado di membership = 1
3. L'altezza di un fuzzy set è rappresentata dal grado massimo di membership definito al suo interno
4. un fuzzy set si dice normale se ha altezza = 1 e subnormale se ha altezza < 1.

Per poter rappresentare le variabili linguistiche però, è necessario definire alcune operazioni di base come negazione, congiunzione e unione
La negazione che rappresenterò con il simbolo (-) deve rispettare alcune proprietà di base come:
1. -0 = 1
2. -1 = 0
3. se x <= y -x >= -y
A queste proprietà possono anche essere aggiunte altre proprietà per definire una negazione stretta o forte.
La negazione viene detta stretta se rispetta le proprietà di continuità e di decrescenza stretta ( se x<y  -x>-y).
La negazione viene detta forte se è stretta e se rispetta la proprietà di involutività (--x = y).
La congiunzione e l'unione vengono definite tramite i concetti di T-norma e T-conorma [0,1]^2 -> [0,1].
In generale, congiunzione e unione devono rispettare le proprietà di identità, commutatività, associatività e transitività.
Per la congiunzione l'identità è data dall'elemento 1, per la disgiunzione l'identità è data dall'elemento 0.

I fuzzy controller permettono di modellare sistemi complessi le cui dinamiche possono sfuggire ad un’analisi matematicamente precisa.
I fuzzy controller hanno un'architettura formata da 4 componenti principali, la fuzzyfication interface, la decision logic, il knowledge base e la defuzzyfication interface.
la fuzzyfication interface permette di passare da un valore crisp ad uno fuzzy. (i crisp derivano dal sensori del controlled sistem come output.)
FUZZIFICATION IS THE PROCESS OF TRANSFORMING A CRISP SET INTO A FUZZY SET.
In pratica da un input crisp utilizzo le membership functions che sono state definite per ogni valore linguistico per modellare la variabile linguistica, per passare dall'input crisp ad un valore di appartenenza per ogni fuzzy set, quindi ad un valore fuzzy, cioè uan rappresentazione su fuzzy sets.
Tipicamente dato un certo termine linguistico esso viene adoperato nelle varie membership functions definite e ciò che si ottiene è un fuzzy set che mapperà tale termine linguistico su valori di appartenenza al fuzzy set.
la decision logic riceve l'input dalla fuzzyfication interface e computa l'output per mezzo della knowledge base.
La knowledge base è formata da due componeneti, il database e il rule base.
il rule base contiene regole del tipi if X then Y, mentre il database contiene la rappresentazione dei fuzzy set utilizzati.
l'intervallo Y che rappresenta l'unione dei fuzzy set selezionati tramite le rules verrà poi mappato ad un valore crisp tramite defuzzification interface.
l'operazione di defuzzyfication avviene tramite 3 operazioni principali: 1. MCM, 2. MOM, 3.COG
1. dato l'intervallo Y viene preso un punto arbitrario y nell'intervallo che ha valore massimo.
2. preso i valori y nell'intervallo Y che hanno valore massimo, viene attuata una media su di essi.
3. viene restituita il centro di gravità dell'intervallo Y.
Esistono due tipi possibili di controllers il Mamdani e il Takagi-Sugeno:
Il Mamdani nelle regole if X then Y opera con fuzzy sets, è pertanto richiesta la fase di defuzzyfication.
Il takagi-sugeno controller, nella fase dedicata alla decision logic, nelle regole if X then f(X) presenta come consequent una funzione sulle varibili di input, di solito lineare, pertanto non necessità della fase di defuzzyfication, ma il valore in output dalla decision logic sarà un valore crisp.

FUZZY CLUSTERING:(small distance to the center of the cluster = high membership, also the opposite is true) parte dal concetto di hard c-means, a.k.a il solito k-means, poi passa a fuzzy clustering, in cui un elemento può avere un certo grado di membership ad ogni cluster, importante evidenziare che permette di risolvere problemi come quando si ha un punto in mezzo a due clusters e non si sa come assegnarlo.
il fuzzy clustering può essere probabilistico o possiblistico.
La differenza sta nelle condizioni imposte sulla funzione di appartenenza al cluster.
Nel caso probabilistico ci si riferisce al all'hard c-means quindi i clusters non sono vuoti e ogni elemento ha lo stesso peso all'interno del cluster, il caso possibilistico elimina una condizione e viene mantenuta solo la prima, cioè i clusters non sono vuoti, garantendo una maggiore libertà e permettendo il fuzzy clustering. il caso possibilistico tratta i dati con una bassa membership function come outliers
clustering validity:
si cerca di capire quanto bene sono stati classificati i punti, questa parte è necessaria e tediosa ma permette di capire se il risultato del clustering è uno tra quelli desiderati, è necessario in quanto si ricorda che il clustering è unsupervised.
Alcuni criteri da ricercare sono: una chiara separazione tra i cluster, minimo volume dei cluster, massimo numero di punti concentrati vicino al centro del cluster. 
Il fuzzy c-means così come il possibilistic c-means utilizzano un concetto di distanza per cercare di definire la somiglianza tra gli elementi, tuttavia essa riconosce solo clusters sferici, per cercare di ovviare a questo problema sono state documentate altre possibili funzioni di distanza per cercare di ottenere altre forme di clusters possibili.
alcune varianti sono: 
fuzzy Gustafson-Kessel algorithm (preferita quando il clustering è fatto per la generazione di fuzzy rules, the shapes of the clusters are more ovals/elliptical than spherical)
fuzzy shell clustering algorithms(particolarmente utile per image recognition)
kernel-based variants (stesso approccio che si ha nelle SVMs o support vector machines, i punti vengono mappati su dimensioni maggiori per cercare di ottenere una separazione più netta e pulita dei punti e quindi classificare corretamenti gli elementi del cluster piuttosto che di un altro.)
RANDOM SET: con il clustering abbiamo descritto con tecniche fuzzy dei dati crisp, adesso con i random set vogliamo descrivere dei dati fuzzy.
attraverso i concetti di random set e random variable possiamo mappare su di essi il concetto di fuzzy set in modo da avere una distribuzione di probabilità che ci permetta di descrivere il nostro fuzzy set, come ad esempio quante persone parlano sia spagnolo che inglese con un grado di 0.8 all'interno del nostro dataset.

l'exstension principle viene utilizzato per estenede le operazioni matematiche all'insieme fuzzy, ad esempio f(X) = x^2. riguarda le funzioni matematiche/aritmetiche che possono quindi essere descritte anche su insiemi fuzzy.


Formally, a membership function for a fuzzy set A on the universe of discourse X is defined as µA: X → [0, 1], where each element of X is mapped to a value between 0 and 1. This value, called membership value or degree of membership, quantifies the grade of membership of the element in X to the fuzzy set A. Here, X is the universal set and A is the fuzzy set derived from X.




un fuzzy set A è definito come un mapping dall'insieme universo X tramite membership function µA: X->[0,1], il valore assegnano ad ogni elemento x appartente a X è chiamato grado di membership e indica il valore di appartenenza dell'elemento x all'insieme fuzzy A.
